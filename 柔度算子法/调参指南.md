# 调参指南

针对柔度拟合任务，如需进一步在模型大小、精度及训练时长方面优化，主要的 **6 个待调超参数** 是：

1. **神经网络结构相关**：`hidden_dim` 隐藏层宽度，`block_num` 隐藏层深度  
2. **学习率相关**：`lr / lr_min` 学习率大小，`use_cosine` 学习率衰减模式  
3. **训练相关**：`batchsize` 显存相关，`epoch_num` 训练轮数  


## 调参整体思路

在 **模型容量、优化难度、计算资源** 之间做整体平衡。

## 一、网络结构相关超参数（决定模型表达潜力）

### 1. `hidden_dim` —— 隐藏层宽度（单层表达能力）

每一层中神经元的数量，直接决定单层的特征表示维度。

**对模型与结果的影响：**

- **hidden_dim 小**
  - 表达能力不足，复杂柔度分布难以拟合
  - 误差下限高（欠拟合）
- **hidden_dim 大**
  - 表达能力增强，能拟合更复杂的非线性关系
  - 参数量和显存、计算量线性增长
  - 更容易过拟合，对学习率更敏感

---

### 2. `block_num` —— 隐藏层深度（特征组合层次）

网络中隐藏层（或 block）的数量，决定非线性组合的层次深度。

**对模型与结果的影响：**

- **block_num 少**
  - 模型接近“浅层函数逼近”
  - 对简单、平滑柔度场有效
- **block_num 多**
  - 能表达更复杂的层级关系（多尺度、局部–全局耦合）
  - 梯度传播变难，训练更依赖学习率策略
  - 单次前向 / 反向计算成本增加

**关键点：**

- 宽度 = 表达“多少种特征”
- 深度 = 表达“特征如何组合”

####  结构参数之间的耦合关系

- **宽而浅**：参数多，但学习相对稳定  
- **窄而深**：参数可能更少，但更难训练  
- **宽 + 深**：表达力最强，但最吃显存、最挑学习率  

在柔度拟合任务中，**深度主要用于表达非线性结构关系，宽度用于表达局部自由度数量**，两者不能只调一个。


## 二、学习率相关超参数（决定模型潜力的开发情况）

### 3. `lr / lr_min` —— 初始学习率与最小学习率

- `lr`：优化器每一步参数更新的步长  
- `lr_min`：学习率衰减后的下限，决定收敛阶段的“微调能力”

**对模型与结果的影响：**

- **lr 过大**
  - loss 震荡、甚至发散
  - 深层网络尤为明显
- **lr 过小**
  - 收敛慢，容易卡在次优解
- **lr_min 太大**
  - 后期无法精细逼近，误差下不去
- **lr_min 太小**
  - 训练时间明显拉长，收益有限

---

### 4. `use_cosine` —— 是否启用余弦退火（Cosine Annealing）学习率调度

该参数用于控制学习率调度策略的类型：

- **不启用余弦退火（`use_cosine = False`）**  
  → 使用默认的余弦衰减策略（cosine decay）
- **启用余弦退火（`use_cosine = True`）**  
  → 使用完整的余弦退火调度（cosine annealing）

两者都会随训练推进降低学习率，但调度行为和对训练阶段的刻画能力不同。


**对模型与最终结果的影响：**

- **默认余弦衰减（未启用余弦退火）**
  - 学习率整体随训练逐步下降  
  - 调度行为相对简单，阶段感不强  
  - 对训练轮数和 `lr` 初值仍较敏感  
  - 适合结构较浅、规模较小的模型  

- **启用余弦退火（Cosine Annealing）**
  - 明确区分训练前期 / 中期 / 后期  
  - 前期保持较大学习率，充分探索参数空间  
  - 后期学习率快速退火到 `lr_min`，进入精细拟合阶段  
  - 收敛更平滑，后期震荡明显减少  
  - 对深层或大宽度模型更友好  


**经验性结论：**  
模型规模越大、结构越复杂，越应该启用余弦退火，而不是仅依赖默认的余弦衰减。


####  学习率策略与网络结构的耦合关系

- 网络 **越深 / 越宽**
  - → 参数空间越复杂  
  - → 训练过程越依赖明确的“探索 → 收敛”阶段划分  
  - → 更适合启用余弦退火  

- **复杂网络 + 仅使用默认余弦衰减**
  - → 后期学习率下降不够“果断”  
  - → 容易出现 loss 下降变慢、震荡或卡平台  

- **大模型 + 余弦退火（`use_cosine = True`）**
  - → 前期不易错过好解  
  - → 后期更容易压低最终误差  
  - → 对初始 `lr` 的容忍度明显更高  


## 三、训练相关超参数（决定训练时间）

### 5. `batchsize` —— 批大小（显存与梯度质量）

每一步反向传播使用的样本数量。

**对模型与结果的影响：**

- **batchsize 小**
  - 梯度噪声大，泛化性好
  - 收敛慢，训练不稳定
- **batchsize 大**
  - 梯度更稳定，训练更快
  - 占用显存多，可能更容易过拟合

---

### 6. `epoch_num` —— 训练轮数（学习是否充分）

完整遍历训练集的次数。

**对模型与结果的影响：**

- **epoch 太少**
  - 模型尚未学完，欠拟合
- **epoch 太多**
  - 若无衰减或正则，易过拟合
  - 时间成本上升

####  `batchsize`、`epoch` 与学习率的耦合

- **batchsize ↑**
  - → 梯度更稳定
  - → 可以适当增大 `lr`
- **batchsize ↓**
  - → 梯度噪声大
  - → 需要更小 `lr` 或更多 `epoch`
- **使用 cosine 衰减**
  - → 通常需要更大的 `epoch_num` 才能完整走完衰减曲线

---

## 调参顺序与思路

1. **先定结构（`hidden_dim` + `block_num`）**  
   → 决定模型“能力上限”
2. **再配学习率策略（`lr` + cosine）**  
   → 保证“学得动、学得稳”
3. **最后用 `batchsize` + `epoch_num` 平衡资源与效率**

**所有问题几乎都不是“某一个参数不对”，而是“耦合关系没匹配”。**

---

## 全部可调参数

| 参数名 | 含义 | 说明 |
|------|------|------|
| `output_dim` | 输出维度索引 | 每次只训练一个输出维度 |
| `hidden_dim` | 隐藏层维度 | 决定网络容量，影响精度与推理速度 |
| `block_num` | 网络 block 数 | 控制模型深度 |
| `epoch_num` | 训练轮数 | 默认 25 |
| `batchsize` | 训练 batch size | 与显存占用直接相关 |
| `lr / lr_min` | 学习率范围 | 与 cosine scheduler 配合使用 |
| `use_cosine` | 是否使用余弦退火 | 默认开启 |
| `use_dropout / dropout_p` | Dropout 配置 | 当前默认关闭 |
| `use_L2 / L2weight` | L2 正则 | 当前默认关闭 |
| `use_standerscale` | 是否使用标准化 | 必须与推理阶段一致 |
