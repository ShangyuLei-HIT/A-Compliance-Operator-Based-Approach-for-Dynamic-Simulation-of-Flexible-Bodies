# GPU 推理性能报告
---
## 一、架构总览
- main()
  - 创建 `Ort::Env` ← 只一次
  - 创建 `SessionOptions`
    - Append `CUDA EP` ← 只一次
  - 加载 `scaler / scale.json` ← 只一次
  - 加载 9 个 `Ort::Session` ← 各一次
  - for（几万次迭代）
    - for（9 个模型）
      - `session[i].Run()` ← 只有这里反复执行

定义一个“推理上下文”结构体，将初始化次数由原来的数十万次改为仅初始化一次，  
避免 GPU 冷启动，被反复开启又关闭。

### 鲁棒性

通过 `bool use_gpu` 控制使用 GPU 或 CPU 进行推理；  
`use_gpu = true` 时，若检查不到 GPU 或 CUDA 环境不匹配，将自动回退至 CPU 推理。

---

## 二、推理速度测试

以 **500 次迭代**、**单次迭代推理 3 万组点对** 为例，分别在 CPU 和 GPU 上进行测试。

---

### GPU RTX 3090（显存占用 5GB / 24GB）

- 初始化时间：0.303723 s  
- 推理时间：25.963877 s  
- 总耗时：26.2676 s  
- 单次推理用时：0.00577 s  
- 单个迭代步用时：0.0519278 s  

可以看到，经过整体架构优化后，ONNX 平台 GPU 推理速度与最初 PyTorch 平台测试结果基本一致。

---

### PyTorch 平台 GPU 推理

| points | time      |
|--------|-----------|
| 10w    | 0.0148s   |
| 3w     | 0.0045s   |
| 3k     | 0.00077s  |

---

### GPU RTX 1650（显存占用 3GB / 4GB）

RTX 1650 由于显存限制，无法执行 3 万点并行，需要每个迭代步分三次进行 1 万点并行推理：

- 初始化时间：0.32495 s  
- 推理时间：1029.14 s  
- 总耗时：1029.465 s  
- 单次推理用时：0.22852 s  
- 单个迭代步用时：2.05668 s  

---

### GPU RTX 3090（显存占用 3GB / 24GB）

RTX 3090 与 RTX 1650 的推理速度相差约 **40 倍**，与显存关系不大，主要源于硬件代际与架构差异。  
在采用与 RTX 1650 相同的推理方式与显存占用（3GB）条件下，RTX 3090 依然快 **30 倍**：

- 初始化时间：0.55176 s  
- 推理时间：34.9868 s  
- 总耗时：35.5386 s  
- 单次推理用时：0.00777 s  
- 单个迭代步用时：0.0699737 s  

由于 RTX 1650 显著落后于当前主流消费级显卡，不建议进一步针对 RTX 1650 进行适配与推理优化。

---

### CPU

- 初始化时间：1.15026 s  
- 推理时间：5133.45 s  
- 总耗时：5134.6 s  
- 单次推理用时：1.141 s  
- 单个迭代步用时：10.2669 s  

---

## 三、性能对比结论

相较于 CPU 推理，优化后的 GPU 推理性能提升约 **200 倍**。  
在单步迭代性能上，相比预期目标 **0.1 s / step**，实际速度提升约 **50%**。

在 **500 次迭代**、**每次 3 万组点对** 的测试条件下，
累计推理规模约 **1500 万组点对**，**总推理耗时为 26 s**。
