# 数据集生成脚本说明(gen_data.py)

## 一、脚本说明

本脚本用于从大型 **PSD / HDF5 模态数据文件** 中生成神经网络训练与推理所需的数据集，主要功能包括：

- 构造节点对输入特征（`nodePairs`）
- 生成对应的标签数据：
  - 位移（`Dis`）
  - 应力（`Stress`）
  - 应变（`Strain`）
- 执行输入与输出的 **归一化 / 标准化** 处理
- 划分训练集与测试集
- 保存推理阶段所需的 `scale` 参数文件

该脚本是整个 **模型训练、评估以及 ONNX 推理流程的数据源头**，其输出结果将被后续所有流程直接依赖。

---

## 二、脚本整体工作流程

数据集生成流程如下所示：

```text
读取 PSD / HDF5 文件
 ├─ 分块读取节点与模态数据（避免内存溢出）
 ├─ 构造节点对输入特征（nodePairs）
 ├─ 计算模态位移 S
 ├─ 计算 Stress / Strain（可选）
 ├─ 随机划分训练集 / 测试集索引
 ├─ 输入特征归一化与标准化
 ├─ 按输出维度拆分标签数据
 ├─ 保存训练集与测试集数据
 └─ 生成推理阶段使用的 scale.json
```

## 三、PSD / HDF5 数据读取逻辑

### 3.1 分块读取函数 `read_dataset_in_chunks()`

为安全、高效地处理超大规模 PSD / HDF5 数据集，脚本中实现了统一的分块读取接口：

```python
def read_dataset_in_chunks(dset, block_rows=1_000_000):
```
作用：

- 分批读取 HDF5 数据  
- 避免一次性加载导致内存溢出  
- 最终统一 reshape 成列向量  

### 3.2 关键数据来源

| 数据 | 含义 |
|----|----|
| `Node/Coordinate` | 节点坐标 |
| `Mode/AttachMode` | 模态位移数据 |
| `Mode/AttachStressMode` | 应力模态 |
| `Mode/AttachStrainMode` | 应变模态（可选） |
| `Mode/AttachModeDofId` | DOF 映射关系 |

## 四、向量化数据构造逻辑

### 采用**并行索引与向量化计算**：

- 整体样本空间按照 **模态数 × 节点数** 一次性展开  
- 使用 `meshgrid` + 展平索引构造完整的 `(i, j)` 组合  
- 通过 `searchsorted` 实现模态节点索引的批量映射，避免逐元素查找  
- 除 `dof_i × dof_j` 的 **3×3 极小循环**外，其余计算全部为向量化操作  

最终生成的数据包括：

- 位移特征 `Dis`（9 维）  
- 应力 / 应变特征 `Stress / Strain`（18 维）  

该设计在大规模样本场景下显著降低了 Python 层循环开销，整体数据生成性能主要受 **NumPy 底层并行能力与内存带宽** 限制。

## 六、输入归一化与标准化

### 6.1 数值缩放

```python
scale_node = 1e1 / max(abs(train_inputs))
```
目的：

- 限制输入量级  
- 减少梯度不稳定问题  

---

### 6.2 标准化（StandardScaler）

```python
scaler = StandardScaler()
train_inputs = scaler.fit_transform(train_inputs)
test_inputs = scaler.transform(test_inputs)
```
并保存：

- `dataset/{name_psd}/scaler_inputs.pkl`

**注意事项：**

- 训练、评估与推理阶段 **必须使用同一个 `scaler`**

## 七、标签生成与缩放

### 7.1 gen_labels() 函数说明

**作用：**

- 生成训练集与测试集标签  
- 对标签数据进行数值缩放，保证不同维度数值量级一致  
- 按输出维度拆分标签并保存为独立文件  
- 生成对应的 `scale_*.json` 参数文件  

---

### 7.2 主要参数说明

| 参数 | 含义 |
|----|----|
| `name_psd` | 数据集名称 |
| `S` | 原始标签矩阵（Dis / Stress / Strain） |
| `Stype` | 标签类型：Dis / Stress / Strain |
| `num` | 输出维度数量 |
| `test_indices` | 测试集索引 |

---

### 7.3 标签缩放策略

```python
scale_S = 1e3 / max(abs(S))
```
目的：

- 将物理量映射到神经网络友好的数值范围  
- 推理阶段必须使用**同一套 scale 参数**  

---

### 输出文件结构

- `dataset/{name_psd}/{Stype}/`
  - `train/`
    - `train_labels{dim}.npy`
    - `train_labels_scaled{dim}.npy`
  - `test/`
    - `test_labels{dim}.npy`
    - `test_labels_scaled{dim}.npy`
  - `scale_{Stype}.json`


## 八、相关参数输出

最终生成的参数文件为：

- `inference/dataset/{name_psd}/scale.json`

内容包括：

| 参数 | 说明 |
|----|----|
| `node_count` | 节点数量 |
| `scale_node` | 输入缩放因子 |
| `scale_Dis` | 位移缩放参数 |
| `scale_Stress` | 应力缩放参数 |
| `scale_Strain` | 应变缩放参数 |

该文件是 **ONNX 推理阶段的关键依赖文件**，必须与训练与数据生成阶段保持严格一致。
